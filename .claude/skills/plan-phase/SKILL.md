# Skill: plan-phase

## Vorgehen

1. Lies `AGENTS.md`, `CLAUDE.md` und passende Datei aus `docs/phases/`.
2. Prüfe aktuellen Stand.
3. Erstelle einen konkreten Plan.
4. Nenne Dateien, die geändert würden.
5. Nenne Tests.
6. Nenne Nicht-Ziele.
7. Warte auf Go.

## Ausgabe

```markdown
# Plan: <Phase>

## Ziel
## Aktueller Stand
## Dateien
## Schritte
## Tests
## Nicht-Ziele
## Risiken
## Definition of Done
```
