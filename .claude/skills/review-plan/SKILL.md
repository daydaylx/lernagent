# Skill: review-plan

## Prüfen

- Ist die Phase klar begrenzt?
- Sind Nicht-Ziele enthalten?
- Werden unnötige Features eingebaut?
- Gibt es Tests?
- Gibt es Sicherheitsrisiken?
- Ist etwas zu schwammig?
- Gibt es konkrete Verifikationskriterien?

## Ausgabe

```markdown
# Plan-Review

## Urteil
## Kritische Probleme
## Unklare Punkte
## Verbesserter Plan
```
