# Project Brief – RepoTutor

## Ein Satz

RepoTutor soll ein privates lokales Tool werden, das Softwareprojekte einliest und Dateien verständlich erklärt.

## Problem

KI-generierter Code ist oft schwer zu verstehen. Der Nutzer möchte nicht blind Änderungen übernehmen, sondern lernen, was in seinen Projekten passiert.

## Ziel

Dateien, Imports, Rollen, Abläufe und Risiken verständlich erklären.

## Nicht-Ziel

RepoTutor soll zuerst nicht selbst Code ändern.

## Geplante Hauptfunktionen

- Projektstruktur erkennen
- Datei auswählen
- Datei einfach erklären
- Zusammenhänge zeigen
- Lernfragen erzeugen
- Rückfragen beantworten
